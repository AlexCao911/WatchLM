# Qwen3-0.6B Stateful-Step KV Int4 Smoke

## Purpose

Try a newer Qwen-class small model on the WatchLM Core ML path, isolated on
branch `codex/qwen-watch-se-runtime`, instead of continuing to tune MiniCPM5-1B
quantization.

## Source

- Model: `Qwen/Qwen3-0.6B`
- Graph: `stateful-step-kv`
- Context: 256
- Layers: 28
- KV heads: 8
- Head dimension: 128
- Tokenizer: `artifacts/hf/Qwen3-0.6B/tokenizer.json`

## Swift Host Load Probe

Command:

```sh
swift run WatchLMBenchmark \
  --runtime coreml \
  --prefill artifacts/coreml/qwen3-0.6b-stateful-step-kv-256-int4/stateful-step-kv-256-int4.mlpackage \
  --tokenizer artifacts/hf/Qwen3-0.6B/tokenizer.json \
  --coreml-graph-interface stateful-step-kv \
  --coreml-load-target prefill \
  --load-only \
  --context 256 \
  --device-profile watch-se-2 \
  --source-model Qwen/Qwen3-0.6B \
  --policy-id qwen3-0.6b-stateful-step-kv-256-int4 \
  --id qwen3-0.6b-load-probe \
  --output artifacts/benchmarks/qwen3-0.6b-stateful-step-kv-256-int4-load-probe.json
```

Observed:

```text

```

This is the first Qwen candidate that clears the practical artifact-size gate
and loads through the Swift/CoreML benchmark harness.

## Swift One-Token Inference Smoke

After adding runtime-candidate graph dimensions and tokenizer special-token
overrides to the Swift benchmark command, the same int4 artifact completed a
real stateful-step inference smoke:

```sh
swift run WatchLMBenchmark \
  --runtime coreml \
  --prefill artifacts/coreml/qwen3-0.6b-stateful-step-kv-256-int4/stateful-step-kv-256-int4.mlpackage \
  --tokenizer artifacts/hf/Qwen3-0.6B/tokenizer.json \
  --coreml-graph-interface stateful-step-kv \
  --coreml-layer-count 28 \
  --coreml-kv-heads 8 \
  --coreml-head-dim 128 \
  --tokenizer-bos-token-id 151643 \
  --tokenizer-eos-token-ids 151645 \
  --context 256 \
  --device-profile watch-se-2 \
  --source-model Qwen/Qwen3-0.6B \
  --policy-id qwen3-0.6b-stateful-step-kv-256-int4 \
  --id qwen3-0.6b-one-token-smoke \
  --prompt-ids en-short-001 \
  --max-new-tokens 1 \
  --allow-missing-references \
  --output artifacts/benchmarks/qwen3-0.6b-stateful-step-kv-256-int4-one-token-smoke.json
```

Observed:

```text
prompt success: 1/1
generated tokens: 1
generated token id: 10810
generated text: "nie"
host load time: 8,188.76 ms
host first token: 9,639.456 ms
host peak RSS: 678.72 MB
thermal: nominal
```

This proves the Qwen int4 artifact can pass through the actual Swift path:

```text
tokenizer -> stateful-step prompt scan -> logits -> sampler -> decode token text
```

It is still a smoke test, not a quality result. The single emitted token is not
meaningful enough to evaluate generation quality, and the Qwen chat template
has not yet been matched against Hugging Face output.
