# Qwen3-0.6B Stateful-Step KV Int4 Smoke

## Purpose

Try a newer Qwen-class small model on the WatchLM Core ML path, isolated on
branch `codex/qwen-watch-se-runtime`, instead of continuing to tune MiniCPM5-1B
quantization.

## Source

- Model: `Qwen/Qwen3-0.6B`
- Graph: `stateful-step-kv`
- Context: 256
- Layers: 28
- KV heads: 8
- Head dimension: 128
- Tokenizer: `artifacts/hf/Qwen3-0.6B/tokenizer.json`

## Remaining Gaps

The result is not yet full Qwen text generation on watch. Remaining work:

1. Swift runtime CLI still defaults stateful-step graph dimensions to MiniCPM
   `24 layers / 2 KV heads / 128 head dim`; Qwen requires
   `28 layers / 8 KV heads / 128 head dim`.
2. The current Swift tokenizer is MiniCPM-named and defaults to MiniCPM BOS/EOS
   IDs. Qwen needs BOS `151643` and EOS `151645`, or tokenizer config driven
   special-token handling.
3. After the graph dimensions and tokenizer special tokens are configurable,
   run a one-token generation smoke, then a short prompt benchmark, then the
   watchOS simulator/device load and latency checks.
