# Stateful KV Smoke Evidence

Date: 2026-05-30

## Scope

This note records the first real Core ML `MLState` smoke test for the Swift
stateful KV runtime path.

It is not a MiniCPM5 quality or memory benchmark. Its purpose is narrower: prove
that the Swift runtime can load a stateful Core ML model, create `MLState`, run
prefill with `prediction(from:using:)`, and reuse the same state through decode.

## Artifact

Generated by:

```text
.venv/bin/python tools/conversion/generate-coreml-smoke-model.py
```

New resources:

```text
Tests/WatchLMCoreTests/Resources/SmokeStatefulKV.mlpackage
Tests/WatchLMCoreTests/Resources/SmokeStatefulKV_macOS.mlmodelc
Tests/WatchLMCoreTests/Resources/SmokeStatefulKV_watchOS.mlmodelc
```

The model has one state buffer:

```text
last_token: fp16[1]
```

The public graph IO is token/logits only:

```text
inputs:  input_ids, position_ids, causal_mask
output:  logits
state:   last_token
```

## Swift Test

`coreMLPrefillDecodeRuntimeRunsStatefulKVGraph` uses:

```text
graphInterface = statefulKV
prefillModelURL == decodeModelURL
decodeTokenInputName = input_ids
decodePositionInputName = position_ids
```

Expected generation:

```text
D, E, F
token IDs: 5, 6, 7
```

## Evidence

Passing command:

```text
swift test --filter coreMLPrefillDecodeRuntimeRunsStatefulKVGraph
```

Observed result:

```text
1 test passed
```

## Limitation

The smoke model validates the Swift `MLState` call chain with a single-token
prefill. A full MiniCPM5 `stateful-kv` artifact still needs separate conversion
and validation for multi-token prefill, 24-layer KV state buffers, and logits
agreement against the PyTorch teacher.
